# 参数配置指南

## 1. 训练参数配置


### 步骤
1.  **拉取代码**：确保你拥有 `config.default.json`。
2.  **创建本地配置**：
    在项目根目录下，复制模板文件并重命名为 `config.json`：
    ```bash
    cp config.default.json config.json
    ```
3.  **修改参数**：
    编辑 `config.json`，根据你的需求调整参数（例如 `hours_to_train`）。

    *此文件已被 `.gitignore` 忽略，你的修改**不会**影响队友。*

---

## 2. 参数加载优先级

`train.py` 启动时按以下顺序加载配置（优先级从高到低）：

1.  **`config.json` (本地私有)** **最高优先级**
    * *用途：* 个人实验、特定机器的定制参数。
2.  **`config.default.json` (仓库默认)**  **中等优先级**
    * *用途：* 团队统一的基准参数，所有人都可见。
3.  **代码默认值** **最低优先级**
    * *用途：* 仅作为文件缺失时的兜底。

因此，每个人只用修改自己本地的 `config.json` 即可，不会影响他人。

---

## 3. 训练结果处理

**严禁上传：**

以下文件体积巨大且频繁变动，**绝对不要**提交到 Git：
* **日志**: `logs/` (TensorBoard 数据)
* **模型**: `models/` (.zip 权重文件)
* **视频**: `videos/` (训练录屏)

**共享方式：**

如果你训练出了最佳模型（如 `best_model.zip`），请通过网盘、HuggingFace 或微信直接发送给队友。
队友下载后，使用 `--load-model` 参数加载即可。
